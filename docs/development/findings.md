# 关键发现

## 业务发现

- 方案覆盖四个阶段：发起与发货、收货与使用、归还与质检、销账闭环。
- V1.1 扩展了自提借样、自提核销和自提归还。
- V1.2 扩展了通用归还入口、虚店聚合商品、FIFO 自动分配、归还批次号。
- 借样入口统一到钉钉助手，但 GMS 仍是调拨单与库存能力的主系统。
- 物流链路存在两类来源：系统自动/外部接口拿到的单号，和人工补录单号。

## 业务约束

- 借样审核要校验借样资格、逾期、数量上限、黑名单商品、库存可用性。
- 自提场景要求 `logisticsMode=2`，且初始状态为待自提。
- 自提核销和归还都支持部分数量，意味着明细层必须有数量累计字段。
- 通用归还按“虚店 + 商品 + 尺码”聚合展示，但最终仍需拆回到具体任务单。
- 一次通用归还提交只能给当前自提样品选择一种归还方式，不支持混合。

## 设计含义

- 需要 `task`、`task_item`、`return_batch`、`return_allocation`、`event_log` 这类分层模型，而不是只保留单一任务表。
- 需要同步接口 + 异步任务双轨架构：同步处理钉钉交互，异步负责 GMS 状态同步、物流同步、唯一码巡检和通知补偿。
- 需要清晰状态机，区分任务主状态、发货状态、收货状态、归还状态、质检状态。

## 环境发现

- `mvn` 已可用，Maven 实际运行在 `Java 26` 上，满足 `Spring Boot 3` 最低要求。
- 项目按 `release 17` 编译，因此代码必须遵守 Java 17 API 边界，不能使用 `List.getFirst()` 这类更高版本 API。
- 当前仓库已经可以执行 `mvn test`，并已通过单元测试。
- 本地 MySQL 权限已修复，`ycf / ycf012!` 可用于 `live_borrow_sample` 的本地 profile。

## 实现发现

- 已完成 `borrow`、`return`、`admin` 三组核心控制器和 DTO。
- 已完成内存版和 MySQL 版数据存储实现，可通过 Spring profile 切换。
- 已完成 FIFO 归还分配领域服务，并通过单元测试验证先进先出逻辑。
- 已完成 MyBatis Mapper、Entity、Flyway migration 和演示种子数据，`local/prod/memory` profile 已可用。
- Testcontainers 集成测试已覆盖 MySQL 申请明细保存、虚店种子数据加载、归还批次分配与任务归还状态持久化。
