# 直播借样 MVP 产品需求与验收标准

## 1. 文档目的

本文件从产品经理视角，明确一期 MVP 的业务目标、角色边界、功能范围、验收标准和上线策略，作为研发、测试、运营和业务接口人的统一基线。

## 2. 产品目标

一期 MVP 只解决四件事：

1. 让借样申请从线下沟通迁移到钉钉统一入口。
2. 让借样、发货、收货、归还、质检、销账形成系统化闭环。
3. 让通用归还支持按虚店聚合和 FIFO 自动匹配任务。
4. 让异常、补偿、审计有据可查。

## 3. 用户角色

### 3.1 借样人

- 发起借样申请
- 查询我的任务
- 快递收货确认
- 自提确认
- 发起单任务归还
- 发起通用归还
- 填写归还物流

### 3.2 货控 / 运营

- 查看全量或所辖任务
- 处理借样异常
- 重试 GMS 同步
- 核对物流与任务

### 3.3 门店 / 仓库

- 查询待自提、待收货、待质检任务
- 确认门店收货
- 确认质检结果

### 3.4 系统管理员

- 维护黑名单规则
- 维护借样额度规则
- 配置钉钉应用参数
- 查看对账、补偿、接口日志

## 4. MVP 功能范围

## 4.1 借样申请

- 钉钉 H5 发起借样
- 支持快递 / 自提
- 自动资格校验
- 自动库存可用性校验
- 自动生成借样任务

## 4.2 履约跟踪

- GMS 借出建单
- 发货状态同步
- 快递签收状态跟踪
- 用户确认收货
- 自提确认

## 4.3 归还闭环

- 单任务归还
- 通用归还
- FIFO 自动分配
- 归还批次生成
- 快递物流补录
- 自提样品自行归还到店
- GMS 归还建单

## 4.4 质检销账

- 仓库收货后转质检
- 唯一码结果查询
- 品相一致自动销账
- 品相降级进入异常池

## 4.5 运营后台

- 任务查询
- 批次查询
- 异常池
- 重试补偿
- 规则配置

## 5. 非目标

一期不做：

- 自动呼叫回程快递
- 赔付流程
- 智能替代商品推荐
- 门店 POS 深度集成
- 复杂 BI 看板

## 6. 用户旅程

## 6.1 借样快递路径

1. 借样人发起借样
2. 系统审核并建借出单
3. 门店发货
4. 借样人确认收货
5. 借样使用中
6. 借样人发起归还
7. 填写物流单号
8. 仓库收货
9. 质检销账
10. 系统通知完成

## 6.2 借样自提路径

1. 借样人发起自提借样
2. 系统审核并建自提单
3. 借样人到店提货并确认
4. 借样使用中
5. 借样人发起归还
6. 选择快递归还或自行归还到店
7. 收货 / 质检 / 销账

## 6.3 通用归还路径

1. 借样人在钉钉点击“发起还样”
2. 选择虚店
3. 查看聚合商品清单
4. 填写归还数量
5. 系统执行 FIFO 分配
6. 生成归还批次号
7. 填写物流或等待门店收货

## 8. 关键业务规则

### 8.1 借样资格

- 用户必须绑定虚店
- 不存在逾期 15 天以上未还任务
- 当前借样中数量不超过额度
- 商品不命中黑名单

### 8.2 自提规则

- 自提必须指定有效门店
- 允许部分提货
- 未全部提完前，任务仍是待自提

### 8.3 归还规则

- 通用归还以“虚店 + SKU + 尺码”聚合展示
- 系统按借出时间升序 FIFO 分配
- 同一次“仅自提样品”归还只能选一种归还方式
- 快递归还必须填写物流单号

### 8.4 质检规则

- 品相一致直接销账
- 借出 A 品归还变 B/C 品进入异常池
- 7 天未获取有效质检结果时进入人工处理

## 9. 验收标准

## 9.1 借样申请

- 借样人可在钉钉完成表单提交
- 权限、额度、黑名单、库存校验可正确拦截
- 成功后生成申请单号、借样主单号、任务单号

## 9.2 借出履约

- GMS 借出建单成功后可回写 GMS 单号
- 快递任务能同步发货状态
- 自提任务能进入待自提状态
- 用户确认收货 / 自提后，任务进入借样中

## 9.3 通用归还

- 可按虚店聚合可还商品
- 归还数量不可超过可还总量
- FIFO 分配结果可追踪到具体任务明细
- 归还批次号生成唯一

## 9.4 物流与收货

- 快递归还可填写物流单号
- 同批次物流能批量关联到任务
- 仓库收货后任务进入质检状态

## 9.5 质检销账

- 唯一码品相一致时任务自动完结
- 品相降级时自动进入异常池
- 用户能收到完成或异常提醒

## 9.6 后台与补偿

- 货控能查询异常池
- 管理员能执行 GMS 重试
- 外部接口调用有日志

## 10. 指标口径

- 借样申请成功率 = 成功提交申请数 / 总提交次数
- GMS 借出建单成功率 = 建单成功任务数 / 建单请求任务数
- 通用归还成功率 = 批次创建成功数 / 批次创建请求数
- 自动销账率 = 自动完结任务数 / 完成归还任务数
- 异常率 = 异常任务数 / 总任务数

### 里程碑 2：归还闭环

### 里程碑 3：质检销账与补偿

## 12. 上线策略

- 先灰度 1 到 2 个直播间
- 先限制虚店范围
- 每日人工复核异常和补偿日志
- 灰度 1 周无严重问题后扩量
